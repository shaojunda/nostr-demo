🔥 Nostr Demo

去中心化社交协议演示应用 - 工作版本

✅ 真实 Nostr 实现

使用 webpack 构建版本，包含完整的 secp256k1 加密和事件验证

解决方案： 请在命令行中运行以下命令：

cd /Users/shaojunda/apps/nervina/nostr-demo
npm run dev

🛠️ 构建说明

由于 nostr-tools 的 CDN 版本存在导入问题，推荐使用 webpack 构建版本。

1. 确认依赖已安装

您已经执行了 npm install，依赖应该已经安装完成。

2. 启动开发服务器

在项目根目录执行：

npm run dev

3. 访问应用

浏览器会自动打开，或者手动访问：

http://localhost:8080

4. 如果仍有问题

可以尝试清理并重新安装：

rm -rf node_modules package-lock.json
npm install
npm run dev

🌟 功能特性

✅ 真实的 secp256k1 加密
使用 nostr-tools 库的真实椭圆曲线加密，确保与所有 Nostr 客户端兼容
✅ 完整的事件验证
包含事件ID计算、签名验证、时间戳验证等完整的 Nostr 协议实现
✅ 标准 NIP 协议支持
支持 NIP-01, NIP-02, NIP-19, NIP-22, NIP-23, NIP-25, NIP-51 等标准协议
✅ 完整的社交功能
创建账号、发布长文、点赞、收藏、关注、评论等完整的去中心化社交功能

🔧 解决的问题

  • 事件ID计算错误：使用标准的 getEventHash() 函数
  • 签名验证失败：使用真实的 secp256k1 ECDSA 签名
  • 中继拒绝事件：完整的事件验证确保协议兼容
  • 编码格式错误：支持标准的 Bech32 编码
